# プロジェクト要約

## 何を作っているか

ロンシール工業の購買部門が、仕入先からの値上げ要請の妥当性をデータで判断するツール（PoC）。

## 今あるもの

Streamlit アプリ。品目を選ぶと以下が表示される：

- 品目基本情報（品名・文書番号・現行価格・最終改定日）
- 原価内訳テーブル（材料費・燃動力費・労務費・梱包費・運賃の時系列比較）
- 推定原価 vs 当社購入価格の折れ線チャート
- 改定履歴（改定日・改定前後の価格・改定理由）
- 提案文作成（デモ）— 交渉方針を原価率に応じて自動生成

## データの流れ

```
サンプルデータ（PDF/Excel 52件）
  ↓ Gemini 2.5 Pro で構造化抽出
個別CSV 3つ（品目マスタ・価格履歴・交渉記録）
  ↓ Python スクリプトで結合（品目コード・日付をキーに紐付け）
統合CSV（77カラム）
  ↓
シミュレーター画面で表示・計算
```

## 取れるデータ / 取れないデータ

### サンプルから取れたもの（AI 抽出済み）

- 品名・文書番号・分類番号
- 成分名（15品目中14品目）
- 含有量%（15品目中12品目）
- 物流情報（経路・所要時間・運送会社）
- 価格改定履歴（改定日・改定前後の金額）
- 改定理由（交渉記録から文脈抽出）

### 外部データが必要（現在ダミー値）

- 市況単価（ガラス原材料・塩ビ樹脂・パルプ・天然ガス・電力 等）
- 労務費
- 運賃

## ダミー値の本番化

### 市況単価（6項目）

- 日銀 CGPI API で「価格指数（変動率）」が取れる
- ただし円/kg の絶対単価ではない
- 運用: クライアントから基準単価をもらい、CGPI 変動率で推定単価を更新
- 例: 基準単価 100円 × CGPI +15% = 推定 115円

### 労務費

- e-Stat API（賃金構造基本統計調査・毎月勤労統計調査）で取得可能

### 運賃

- 国交省告示「標準的な運賃」（令和6年3月改定）PDF から手動取り込み
- API なし。ただし改定頻度が低いので一度取り込めば当面更新不要

### クライアント提示 URL について

クライアントから提示された参照 URL（gdfreak.com、kanpura.net 等）は全て二次ソース。

| クライアント提示 | 実態 | 一次ソース |
|---------------|------|----------|
| gdfreak.com（5件） | 日銀データをグラフ化して表示 | 日銀 CGPI API |
| kanpura.net | 厚労省データを表にまとめたもの | e-Stat API |
| logi-biz.com | 独自調査の有料販売（¥11,000） | 国交省告示（無料） |
| 経産省ページ | 一次ソースだが数量中心 | e-Stat API |

## AI 抽出の信頼度

### 高（確実に取れる）

- 表形式で整理されている（Excel の価格改定履歴等）
- 定型フォーマットの PDF（品質仕様書の組成欄等）
- デジタル生成された PDF

### 中（取れるが確認が必要）

- 自由記述から特定キーワードを拾う（交渉記録の改定理由等）
- フォーマットが書類ごとに異なる（各社見積書）
- 手書きだが字が丁寧で文字数が少ないもの

### 低（取れない or 信頼できない）

- 手書き文字・注記（OCR 精度が不十分）
- かすれ・低解像度スキャン
- 書類に記載がない情報
- 印鑑と文字が重なっている箇所
